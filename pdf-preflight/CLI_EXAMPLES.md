# CLI Usage Examples

## Basic Usage

### Example 1: Simple Validation
Validate a PDF with default settings:

```bash
java -jar pdf-preflight-1.0.0.jar --input document.pdf
```

**Expected Output:**
```
========================================
  PDF Preflight Result
========================================

Status: PASS ✓
Total Pages: 50
Mismatches: 0

Processing Time: 342 ms
JSON Report: preflight-report.json
Text Report: preflight-report.txt
```

**Exit Code:** 0 (PASS)

---

### Example 2: Custom Report Paths
Specify custom output paths for reports:

```bash
java -jar pdf-preflight-1.0.0.jar \
  --input large-document.pdf \
  --output-json /tmp/reports/preflight.json \
  --output-text /tmp/reports/preflight.txt
```

**Exit Code:** 0 (PASS) or 1 (FAIL)

## Advanced Options

### Example 3: Use MediaBox Instead of CropBox
Some PDFs may not have CropBox defined. Use MediaBox for measurements:

```bash
java -jar pdf-preflight-1.0.0.jar \
  --input document.pdf \
  --use-mediabox
```

**Use Case:** PDFs generated by certain tools that don't set CropBox.

---

### Example 4: Custom Tolerance
Adjust the tolerance for dimension comparison (in points):

```bash
# Loose tolerance (0.1 points)
java -jar pdf-preflight-1.0.0.jar \
  --input document.pdf \
  --tolerance 0.1

# Tight tolerance (0.001 points)
java -jar pdf-preflight-1.0.0.jar \
  --input document.pdf \
  --tolerance 0.001
```

**Use Case:** Some PDF generators create tiny variations. Adjust tolerance accordingly.

**Note:** 1 point = 1/72 inch. Default tolerance is 0.01 points (~0.00014 inches).

---

### Example 5: Enable Rasterization
Render failed pages as images using MuPDF:

```bash
java -jar pdf-preflight-1.0.0.jar \
  --input document.pdf \
  --rasterize \
  --raster-dpi 300
```

**Requirements:**
- MuPDF must be installed (`mutool` command available)
- Creates `rasterized-pages/` directory with PNG images

**Exit Code:** Still 0 or 1 (rasterization doesn't affect pass/fail)

---

### Example 6: Custom MuPDF Path
Specify custom path to mutool executable:

```bash
java -jar pdf-preflight-1.0.0.jar \
  --input document.pdf \
  --rasterize \
  --mutool-path /usr/local/bin/mutool
```

**Use Case:** MuPDF installed in non-standard location.

---

### Example 7: Rasterize Specific Pages
Choose which pages to rasterize:

```bash
# Rasterize specific pages
java -jar pdf-preflight-1.0.0.jar \
  --input document.pdf \
  --rasterize \
  --raster-pages 25,87,102

# Rasterize with custom output directory
java -jar pdf-preflight-1.0.0.jar \
  --input document.pdf \
  --rasterize \
  --raster-pages 10,20,30 \
  --output-dir /tmp/rasterized
```

**Default Behavior:** If `--raster-pages` is not specified, all mismatched pages are rasterized.

## CI/CD Integration

### Example 9: Jenkins/GitLab CI
Integrate into CI/CD pipeline:

```bash
#!/bin/bash
java -jar pdf-preflight-1.0.0.jar \
  --input "${BUILD_DIR}/final-document.pdf" \
  --output-json "${WORKSPACE}/reports/preflight.json" \
  --output-text "${WORKSPACE}/reports/preflight.txt"

EXIT_CODE=$?

if [ $EXIT_CODE -eq 0 ]; then
    echo "✅ PDF validation passed"
elif [ $EXIT_CODE -eq 1 ]; then
    echo "❌ PDF validation failed - check reports"
    cat "${WORKSPACE}/reports/preflight.txt"
    exit 1
else
    echo "⚠️  PDF validation error"
    exit 2
fi
```

## Batch Processing

### Example 11: Process Multiple Files
```bash
#!/bin/bash

PDF_DIR="/path/to/pdfs"
REPORT_DIR="/path/to/reports"

for pdf in "${PDF_DIR}"/*.pdf; do
    filename=$(basename "$pdf" .pdf)
    echo "Validating: $pdf"
    
    java -jar pdf-preflight-1.0.0.jar \
      --input "$pdf" \
      --output-json "${REPORT_DIR}/${filename}.json" \
      --output-text "${REPORT_DIR}/${filename}.txt"
    
    exit_code=$?
    
    if [ $exit_code -eq 0 ]; then
        echo "  ✅ PASS"
    elif [ $exit_code -eq 1 ]; then
        echo "  ❌ FAIL - see reports"
    else
        echo "  ⚠️  ERROR"
    fi
    
    echo ""
done
```

## Best Practices

1. **Always check exit codes** in scripts and CI/CD pipelines
2. **Use absolute paths** for input and output files
3. **Set appropriate tolerance** based on your PDF generation workflow
4. **Enable rasterization** for visual inspection of failed pages
5. **Archive reports** for audit trails
6. **Test with sample PDFs** before running on production files
7. **Monitor memory usage** for very large files (>1GB)

## Troubleshooting

### Issue: "Command not found: java"
**Solution:** Install Java 11 or higher
```bash
# macOS
brew install openjdk@11

# Ubuntu
sudo apt-get install openjdk-11-jdk
```

### Issue: "OutOfMemoryError"
**Solution:** Increase heap size
```bash
java -Xmx512m -jar pdf-preflight-1.0.0.jar --input large.pdf
```

### Issue: "MuPDF not available"
**Solution:** Install MuPDF or disable rasterization
```bash
# macOS
brew install mupdf-tools

# Ubuntu
sudo apt-get install mupdf-tools

# Or simply don't use --rasterize flag
```
